   - This is the standard Natural Earth 110m resolution topojson from Mike Bostock's world-atlas package

3. Create `src/lib/map-utils.ts` with these exports:

   a. `ISO_NUMERIC_TO_ALPHA3`: A mapping object from ISO 3166-1 numeric codes (string, e.g. "380") to ISO alpha-3 codes (e.g. "ITA"). The world-atlas topojson uses numeric IDs, but our ScoredCountry data uses alpha-3. This bridge map must cover at least the 163 GPI countries. Build this from the standard ISO 3166-1 list. Use string keys (the topojson IDs are strings).

   b. `safetyColorScale(score: number): string` — Takes a score from 1-10 and returns a CSS color string. Use `d3.scaleLinear` with domain [1, 5.5, 10] mapping to the three design token colors:
      - Score 1 (most dangerous) -> `oklch(0.55 0.20 25)` (--color-danger, red)
      - Score 5.5 (moderate) -> `oklch(0.85 0.15 90)` (--color-moderate, yellow)
      - Score 10 (safest) -> `oklch(0.65 0.15 250)` (--color-safe, blue)
      Use d3.interpolateRgb as the interpolation method (OKLCH not natively supported by d3.scaleLinear, so convert the three OKLCH anchors to hex/rgb literals for the scale). The three anchor hex values:
      - danger (1): #9e3a2a
      - moderate (5.5): #d4b83c
      - safe (10): #4a7fbf

   c. `UNSCORED_COLOR`: Export constant string `"#9ca3af"` (Tailwind gray-400, neutral for unscored countries)

   d. `UNSCORED_COLOR_DARK`: Export constant string `"#4b5563"` (Tailwind gray-600, for dark mode)

   e. `getCountryColor(iso3: string, scoreMap: Map<string, number>, isDark: boolean): string` — Looks up the iso3 in scoreMap, returns safetyColorScale(score) if found, else returns UNSCORED_COLOR or UNSCORED_COLOR_DARK based on isDark.

   f. `type MapCountryData = { iso3: string; name: string; score: number | null; color: string }` — Type for processed country data used by the map component.

   **HTML structure:**
   ```html
   <div id="safety-map-container" class="relative w-full" style="height: calc(100vh - 8rem);"
   ```

   **Frontmatter (server-side):**
   - Accept `lang` and `scores` props
   - Build `scoresData` as a simplified array: `scores.map(c => ({ iso3: c.iso3, name: c.name[lang], score: c.score }))` to minimize JSON payload
   - Build `translations` object with nodata/zoomIn/zoomOut labels from `useTranslations(lang)`
   - Compute `zoomInLabel`, `zoomOutLabel`, `legendSafe`, `legendDanger` from translations

   **Client-side `<script>` block** (using `type="module"`):
   This script runs in the browser. It must:

   a. Import D3 and topojson-client (these will be bundled by Astro/Vite):
      ```js
      import * as d3 from 'd3';
      import * as topojson from 'topojson-client';
      ```

   b. Import map utilities:
      ```js
      import { safetyColorScale, UNSCORED_COLOR, UNSCORED_COLOR_DARK, ISO_NUMERIC_TO_ALPHA3 } from '../lib/map-utils';
      ```

   c. Fetch the topojson (use `fetch('/world-topo.json')` -- the file must be in `public/` or imported). Actually, since `src/data/world-topo.json` won't be in public by default, the script should use a static import or the data should be copied. **Decision: Copy world-topo.json to `public/world-topo.json` during Task 1 as well, OR use Astro's asset handling.** Best approach: place the file at `public/world-topo.json` so it's served as a static asset and fetched client-side. Update Task 1's file path accordingly -- Task 1 should save to BOTH `src/data/world-topo.json` (for potential build-time use) AND `public/world-topo.json` (for client-side fetch).    d. Read data attributes from the container:
      ```js
      const container = document.getElementById('safety-map-container');
      const lang = container.dataset.lang;
      const scores = JSON.parse(container.dataset.scores);
      const translations = JSON.parse(container.dataset.translations);
      ```

   e. Build a `Map<string, {name: string, score: number}>` from scores array keyed by iso3.

   f. Detect dark mode: `document.documentElement.classList.contains('dark')`. Also observe dark mode changes with a MutationObserver on `document.documentElement` watching `class` attribute, and re-color countries when dark mode toggles.

   g. Create SVG with `d3.select(container).append('svg')` at 100% width/height of the container. Set viewBox for responsive sizing.

   h. Use `d3.geoNaturalEarth1()` projection (good for choropleth), fit to container dimensions. The projection choice is Claude's discretion per CONTEXT.md -- Natural Earth 1 is the standard for world choropleths.

   i. Create a `<g>` group for all country paths. Use `topojson.feature()` to convert topojson to geojson features.

   j. For each country feature:
      - Look up iso3 via `ISO_NUMERIC_TO_ALPHA3[feature.id]`
      - Look up score data from the scores map
      - Fill color: if scored, use `safetyColorScale(score)`; if unscored, use UNSCORED_COLOR / UNSCORED_COLOR_DARK
      - Stroke: `#e5e7eb` in light mode, `#374151` in dark mode (sand-200-ish / sand-700-ish)
      - Stroke-width: 0.5
      - CSS cursor: `pointer` if scored, `default` if unscored
      - Add class `country-path` for styling

   k. **Zoom behavior:** Use `d3.zoom()` with:
      - scaleExtent: [1, 8]
      - translateExtent based on SVG dimensions (prevent panning beyond map)
      - On zoom event: apply transform to the `<g>` group
      - Enable wheel zoom, pinch zoom (touch), and drag to pan
      - Wire +/- buttons: `zoom.scaleBy(svg.transition().duration(300), 1.5)` for +, `1/1.5` for -

   l. **Tooltip behavior:**
      - On `mouseenter` (or `touchstart`) on a country path:
        - If scored: show tooltip with `"{name} -- {score}"` format
        - If unscored: show tooltip with `"{name} -- {translations.nodata}"` text
        - Position tooltip near cursor (offset 10px right, 10px below)
      - On `mousemove`: update tooltip position
      - On `mouseleave` (or `touchend`): hide tooltip
      - Tooltip element: use the `#map-tooltip` div, toggle `hidden` class

   m. **Touch behavior for mobile:**
      - On touch devices, first tap shows tooltip (like hover)
      - Second tap on same country (or tap on tooltip) navigates
      - Tap on different country shows that country's tooltip
      - Tap on empty area dismisses tooltip
      - Implementation: track `lastTappedIso3` state variable. On touchstart on a country: if iso3 === lastTappedIso3, navigate; else show tooltip and set lastTappedIso3.

   n. **Click navigation (desktop):**
      - On click on a scored country: navigate to `/{lang}/country/{slug}` where slug is the country name lowercased, spaces replaced with hyphens (e.g., "United States" -> "united-states"). Use iso3 as slug for simplicity and URL stability: `/{lang}/country/{iso3.toLowerCase()}`
      - Actually, per the routes config: `/{lang}/{routes[lang].country}/{iso3.toLowerCase()}`
      - So for English: `/en/country/ita`, for Italian: `/it/paese/ita`
      - Unscored countries: no click action
      - On click: `window.location.href = url`

   o. **Responsive:** On window resize, recalculate projection fitSize and re-render. Debounce resize handler at 250ms.

   p. **Dark mode observer:** Watch `document.documentElement` for class changes. When dark mode toggles, update:
      - Country fill colors (unscored countries change between UNSCORED_COLOR and UNSCORED_COLOR_DARK)
      - Stroke colors
      - Scored countries keep their safety colors (those don't change with theme)
