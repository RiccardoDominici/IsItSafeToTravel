Create `src/pipeline/calibration/build-reference.ts` that:

1. Reads the latest raw data files using `findLatestCached()` from `../utils/fs.js`:
   - `gpi-parsed.json` -> extract `gpi_overall` per country (480 indicators, 160 countries with 3 indicators each; filter for `indicatorName === 'gpi_overall'`)
   - `inform-parsed.json` -> extract all 5 INFORM indicators per country and average them: `inform_natural`, `inform_health`, `inform_epidemic`, `inform_governance`, `inform_climate`
   - `worldbank-parsed.json` -> extract `wb_political_stability` per country

2. Build a composite reference score (1-10) for each country using weighted average of available sources:
   - GPI weight 0.50: Scale 1-5 (lower = safer). Formula: `(5 - gpi_overall) / 4 * 9 + 1`
   - INFORM weight 0.30: Average of 5 INFORM indicators (0-10, lower = safer). Formula: `(10 - inform_avg) / 10 * 9 + 1`
   - WB Political Stability weight 0.20: Scale -2.5 to 2.5 (higher = better). Formula: `(wb_ps + 2.5) / 5 * 9 + 1`
   - When a source is missing for a country, redistribute its weight proportionally among available sources
   - Exclude countries with zero available sources

3. Clamp all reference scores to [1, 10]

4. Write output as `src/pipeline/calibration/reference-scores.json`:
```json
{
  "generatedAt": "ISO 8601 timestamp",
  "sourceData": "gpi-parsed.json + inform-parsed.json + worldbank-parsed.json",

}
```

<task type="auto">
  <name>Task 2: Compute deviation report between site scores and reference scores</name>
  <files>src/pipeline/calibration/calibrate.ts, data/calibration/deviation-report.json</files>
  <read_first>

  <action>
Create `src/pipeline/calibration/calibrate.ts` that:

1. Loads site scores from `loadLatestSnapshot()` (from `../scoring/snapshot.js`)
2. Loads reference scores from `./reference-scores.json`
3. For each country present in BOTH datasets, computes:
   - `delta = siteScore - referenceScore` (positive = site rates safer than reference)
   - `absDelta = Math.abs(delta)`
4. Assigns each country a UN region using a static `REGION_MAP: Record<string, string>` defined in the file. Map ISO3 codes to 5 regions: Africa, Americas, Asia, Europe, Oceania. Must cover at least all countries present in both datasets. Use comprehensive mapping (~200 entries).
5. Assigns risk tier based on reference score: safe (7-10), moderate (4-7), dangerous (1-4)
6. Assigns geopolitical bloc using a static `BLOC_MAP: Record<string, string>` with groups: NATO, EU, BRICS, G7, Non-aligned. Countries can appear in multiple blocs -- use primary affiliation.

7. Computes summary statistics:
   - `meanAbsoluteDeviation` (MAD): average of all |delta|
   - `meanSignedDeviation`: average of all delta (reveals systematic bias direction)
   - `stdDev`: standard deviation of deltas
   - `pearsonR`: Pearson correlation coefficient between site scores and reference scores
   - `totalCountries`: number of countries in comparison

8. Writes output to `data/calibration/deviation-report.json`:
```json
{
  "generatedAt": "ISO 8601",
  "snapshotDate": "2026-03-27",
  "weightsVersion": "7.0.0",
  "summary": {
    "totalCountries": 160,
    "meanAbsoluteDeviation": 0.85,
    "meanSignedDeviation": 0.12,
    "stdDev": 1.05,
    "pearsonR": 0.88
  },
  "countries": [
    { "iso3": "AFG", "name": "Afghanistan", "siteScore": 2.05, "referenceScore": 1.52, "delta": 0.53, "absDelta": 0.53, "region": "Asia", "tier": "dangerous", "bloc": "Non-aligned" },
    ...
  ]
}
```

9. Sorts countries array by absDelta descending (largest deviations first)
10. Prints a summary table to stdout showing top 10 outliers and summary stats
11. Creates `data/calibration/` directory if it doesn't exist (use `mkdirSync` with `recursive: true`)
12. Run as standalone: `npx tsx src/pipeline/calibration/calibrate.ts`
  </action>
