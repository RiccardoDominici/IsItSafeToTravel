1. `fetchBeAdvisories` (HTML-09 Belgium):
   - Fetch `https://diplomatie.belgium.be/fr/pays` to get country links
   - Parse `<a>` elements linking to `/fr/pays/{slug}` to build country list
   - Match country names via `getCountryByName()` (try French names from COUNTRIES)

   - Use `normalizeBeLevel(pageText)` on each per-country page body text

   - Sample first 80 countries to avoid excessive requests
   - Use `normalizeDkLevel(pageText)` on body text
   - indicatorName: `advisory_level_dk`, source: `advisories_dk`

3. `fetchSgAdvisories` (HTML-11 Singapore):
   - Fetch `https://www.mfa.gov.sg/Overseas-Singaporeans/Travelling-Overseas/travel-advisories-notices-and-visa-information`
   - Parse listing page for country entries, look for category text ("Travel page with travel advisory", "Travel page with travel notice", "Travel page")
   - Map categories: "advisory" -> level 3, "notice" -> level 2, plain -> level 1
   - For advisory/notice countries, optionally batch-crawl per-country pages to refine with `normalizeSgLevel(pageText)`
   - indicatorName: `advisory_level_sg`, source: `advisories_sg`
   - Handle pagination if present (check for next-page links)

4. `fetchRoAdvisories` (HTML-12 Romania):
   - Try `https://mae.ro/en/travel-alerts` and `https://www.mae.ro/en/travel-alerts`
   - IMPORTANT: Site returned 503 in research -- handle gracefully, return empty if unavailable
   - If page loads, parse alert entries for country mentions and severity
   - Use text-pattern matching to find country names (via `getCountryByName`)
   - Use `normalizeRoLevel()` if numeric level found, otherwise infer from text keywords
   - Event-based: only produce entries for countries explicitly mentioned in alerts
   - indicatorName: `advisory_level_ro`, source: `advisories_ro`

5. `fetchRsAdvisories` (HTML-13 Serbia):
   - Fetch `https://www.mfa.gov.rs/en/citizens/travel-abroad/visas-and-states-travel-advisory`
   - Parse listing page for country card links
   - Batch-crawl per-country pages (concurrency 3) and use `normalizeRsLevel(pageText)`
   - indicatorName: `advisory_level_rs`, source: `advisories_rs`

6. `fetchEeAdvisories` (HTML-14 Estonia):
   - Fetch `https://reisitargalt.vm.ee/` listing page
   - Parse alphabetical country links to extract Estonian slugs dynamically (do NOT hardcode)
   - Map country names back to ISO3 via `getCountryByName()` (try both Estonian text from links and English name)
   - Batch-crawl per-country pages at `https://reisitargalt.vm.ee/sihtkoht/{slug}/` (concurrency 3)
   - Use `normalizeEeLevel(pageText)` on page body
   - indicatorName: `advisory_level_ee`, source: `advisories_ee`

7. `fetchHrAdvisories` (HTML-15 Croatia):
   - Try `https://mvep.gov.hr/services-for-citizens/travelling-abroad/travel-warnings/22718`
   - If page loads, parse for country mentions and advisory text
   - IMPORTANT: Site is JS-heavy, cheerio may return minimal content. If `$('a').length < 10`, return empty gracefully
   - Use `normalizeHrLevel(pageText)` for any text found
   - indicatorName: `advisory_level_hr`, source: `advisories_hr`
   - This is expected to be the weakest source -- accept sparse/empty results

8. `fetchArAdvisories` (HTML-16 Argentina):
   - Fetch `https://www.cancilleria.gob.ar/es/servicios/viajar-al-exterior/alertas`
   - Parse alert list items for country mentions (Spanish text)
   - Match country names using Spanish names from COUNTRIES: `country.name.es`
   - Use `normalizeArAlert(alertText)` for each alert
   - Event-based: only produce entries for countries explicitly mentioned
   - indicatorName: `advisory_level_ar`, source: `advisories_ar`
