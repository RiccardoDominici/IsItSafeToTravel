The bug: When user clicks a dropdown item, the input's `blur` event fires first, which uses `setTimeout(() => dropdown.classList.add('hidden'), 200)` to hide the dropdown. The 200ms delay is a race condition -- sometimes the dropdown hides before the `click` event fires on the item.

Fix: Change dropdown items from `click` to `mousedown` event listener, and use `relatedTarget` in the blur handler.

**In src/pages/en/compare.astro:**

1. Line 195: Change `el.addEventListener('click', () => {` to `el.addEventListener('mousedown', (e) => { e.preventDefault();`
   The `e.preventDefault()` prevents the mousedown from stealing focus, so the blur handler's `relatedTarget` check works and the selection fires reliably.

2. Lines 255-258: Replace the blur handler:
   ```
   input.addEventListener('blur', () => {
     setTimeout(() => dropdown.classList.add('hidden'), 200);
   });
   ```
   With:
   ```
   input.addEventListener('blur', (e) => {
     const related = (e as FocusEvent).relatedTarget as Node | null;
     if (related && dropdown.contains(related)) return;
     dropdown.classList.add('hidden');
   });
   ```
   This hides the dropdown immediately on blur UNLESS focus moved to a dropdown item. No more setTimeout race condition.

**In src/pages/it/confronta.astro:**
- Apply the exact same two changes at the same line numbers (195 and 255-258).
