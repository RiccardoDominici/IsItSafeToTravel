## Human Verification Required

### 1. Map Click Navigation

**Test:** Start `npx astro dev`, visit `http://localhost:4321/en/`, click any country on the map.
**Expected:** Browser navigates to `/en/country/{iso3}` (e.g., `/en/country/ita`) and the detail page loads without 404, displaying all 6 sections.
**Why human:** `getStaticPaths` only generates pages when `data/scores/latest.json` is present. If no score data exists, no pages are generated and every click results in a 404. This must be verified with actual pipeline output data.

### 2. Italian Locale Rendering

**Test:** Visit `http://localhost:4321/it/`, click the same country, observe the page at `/it/paese/{iso3}`.
**Expected:** All section headings, pillar names, advisory labels, summary text, and back-to-map link are in Italian.
**Why human:** Translation key correctness and natural Italian language quality cannot be verified programmatically.

### 3. hreflang Tag Correctness

**Test:** View source of a rendered country detail page (e.g., `curl http://localhost:4321/en/country/ita` after build).
**Expected:** `<head>` contains `<link rel="alternate" hreflang="en" href="...en/country/ita">` and `<link rel="alternate" hreflang="it" href="...it/paese/ita">`.
**Why human:** `getAlternateLinks` path swapping for locale-prefixed nested routes (e.g., `/en/country/ita` → `/it/paese/ita`) requires confirming the i18n utility correctly handles this pattern.

### 4. Zero Client-Side JavaScript for Charts

**Test:** Open browser DevTools → Network tab → filter by JS → load a country page.
**Expected:** No JavaScript bundles loaded for chart rendering. SVG charts are static inline HTML.
**Why human:** The D3 import in `.astro` frontmatter should be build-time only, but this requires runtime network inspection to confirm no client JS ships for charts.

### 5. Dark Mode and Mobile Responsiveness

**Test:** On a country detail page, toggle dark mode (via the header toggle). Then resize browser to 375px width.
**Expected:** (a) All text readable with appropriate contrast in dark mode. (b) ScoreHero stacks vertically, PillarBreakdown bars fit within mobile viewport, all sections readable without horizontal scroll.
**Why human:** Visual appearance, contrast, and layout correctness require direct inspection.
