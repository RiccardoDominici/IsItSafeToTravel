**Client script — full rewrite:**
Replace the existing `<script>` block entirely. The new script must:

1. **Import D3 modules:** `import { scaleLinear, scaleTime, line, area, utcFormat, brushX, select } from 'd3';`
   Also import scoreToColor: `import { scoreToColor } from '../../lib/colors';`

2. **Parse data:** Read `data-history` attribute as `HistoryPoint[]`, read `data-lang` for locale.

3. **Render function** `renderChart(filteredData, fullData)`:
   - Chart dimensions: width=800, height=240, margin={top:20, right:20, bottom:40, left:50}
   - Compute innerWidth, innerHeight from dimensions and margin
   - Create x scale: `scaleTime().domain([min date, max date]).range([0, innerWidth])`
   - Create y scale: `scaleLinear().domain([1, 10]).range([innerHeight, 0])`
   - Clear the SVG content, then build:
     - `<defs>` with linearGradient id="trendGradient" (same stops: #c47a5a at 0% opacity 0.3, at 100% opacity 0.05)
     - `<defs>` with `<clipPath id="chart-clip"><rect x="0" y="0" width="${innerWidth}" height="${innerHeight}"/></clipPath>`
     - Main `<g>` with transform translate(margin.left, margin.top)
     - Inside main g: Y gridlines at scores [2,4,6,8,10] with stroke="currentColor" stroke-width="0.5" stroke-dasharray="4 4" opacity="0.3"
     - Area fill path with clip-path="url(#chart-clip)" fill="url(#trendGradient)"
     - Trend line path with clip-path="url(#chart-clip)" stroke="currentColor" stroke-width="2"
     - Endpoint circle at last data point with fill from scoreToColor
     - Y axis labels at [-10, tick.y] for scores [2,4,6,8,10], class="text-xs fill-sand-500 dark:fill-sand-400" font-size="12"
     - X axis labels (5-6 evenly spaced), positioned at [x, innerHeight+25], using utcFormat('%b %d'), class="text-xs fill-sand-500 dark:fill-sand-400" font-size="11"
   - Use `svg.innerHTML = ...` string building approach (matching compare page pattern)

4. **Brush setup** — after initial render:
   - Create a brush group: append a `<g class="brush">` to the main chart group
   - Use `brushX().extent([[0, 0], [innerWidth, innerHeight]])`
   - On brush "end" event:
     - If no selection (click without drag), ignore
     - Get [x0, x1] pixel range from selection
     - Convert to dates via x.invert()
     - Filter fullData to points within date range
     - If fewer than 2 points in range, ignore
     - Call renderChart(filteredPoints, fullData) with the filtered data
     - Show the reset button by removing 'hidden' class
     - Clear the brush selection after zoom (call `select(brushGroup).call(brush.move, null)`)
   - Style the brush overlay: `.selection { fill: #c47a5a; fill-opacity: 0.15; stroke: #c47a5a; stroke-opacity: 0.4; }`
   - Re-attach brush after each renderChart call

5. **Reset button handler:**
   - On click: call renderChart(fullData, fullData), hide the reset button
   - Use 300ms ease transition: wrap the renderChart call so that the SVG paths transition smoothly. Apply transition by:
     - Before re-render, store current path d attributes
     - After re-render, use `select(pathElement).attr('d', oldD).transition().duration(300).ease(d3.easeCubicOut).attr('d', newD)` pattern
     - Import `easeCubicOut, transition` from d3 for this
   - Also apply 300ms transition when zooming in (same pattern)

6. **Tooltip** — preserve existing behavior:
   - On mousemove over SVG: find nearest data point by x position, show tooltip with date + score
   - Date format: `new Date(point.date + 'T00:00:00Z').toLocaleDateString(lang === 'it' ? 'it-IT' : 'en-US', { month: 'short', day: 'numeric', year: 'numeric' })`
   - Position tooltip above the nearest point
   - On mouseleave: hide tooltip
   - On touchstart (passive: false, preventDefault): same as mousemove logic
   - On touchstart outside container: hide tooltip
   - Tooltip must work correctly on the zoomed (filtered) data, not the full data

7. **Initial call:** `renderChart(fullData, fullData)` on DOM ready.

**Important implementation notes:**
- ALL date parsing MUST use `new Date(d.date + 'T00:00:00Z')` (Phase 11 fix, non-negotiable)
- Use utcFormat('%b %d') for axis tick labels (Phase 11 fix)
- The brush must be re-attached after every renderChart call since innerHTML clears the SVG
- Do NOT use d3.select().append() for main chart structure — use innerHTML string building (matches compare page pattern, simpler)
- DO use d3.select() for brush attachment and transitions (brush requires D3 selection)
- Keep the Astro frontmatter d3 imports (scaleLinear, scaleTime, etc.) only if still needed for showChart logic; remove unused imports
