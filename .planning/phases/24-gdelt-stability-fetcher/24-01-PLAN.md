  <action>
Create `src/pipeline/fetchers/gdelt.ts` that exports `fetchGdelt(date: string): Promise<FetchResult>`.

**API details:**
- Endpoint: `https://api.gdeltproject.org/api/v1/dash_stabilitytimeline/dash_stabilitytimeline`
- Query params: `LOC={fips}&VAR=instability&OUTPUT=csv&TIMERES=day&SMOOTH=3`
- Returns CSV with columns: date, instability_ratio (0-1 range, higher = more unstable)
- We want the LATEST row's instability value for each country

**Implementation steps:**

1. Import `FIPS_TO_ISO3` from `../config/fips-to-iso3.ts` and `getCountryByIso3` from `../config/countries.ts`
2. Import `Papa` from `papaparse` for CSV parsing
3. Import `writeJson, readJson, getRawDir, findLatestCached` from `../utils/fs.ts`
4. Import `join` from `node:path`

5. Create helper `async function fetchCountryInstability(fips: string): Promise<{fips: string; value: number} | null>`:
   - Build URL with the FIPS code as LOC param
   - Fetch with `AbortSignal.timeout(10_000)` (10s per country, not 30s)
   - Parse response text as CSV using `Papa.parse(text, { header: false, skipEmptyLines: true })`
   - Take the LAST row (most recent day), column index 1 (instability value)
   - Parse as float. If NaN or no rows, return null
   - Return `{ fips, value }` on success, null on error
   - Catch errors silently (log warning), return null -- individual country failures should not crash the fetcher

6. Main `fetchGdelt` function:
   - Get all FIPS codes: `Object.keys(FIPS_TO_ISO3)`
   - Filter to only FIPS codes whose ISO3 maps to a valid country via `getCountryByIso3` (skip territories/islands not in our country list)
   - Fetch countries SEQUENTIALLY with a 500ms delay between requests (`await new Promise(r => setTimeout(r, 500))`) to respect undocumented rate limits
   - Log progress every 25 countries: `[GDELT] Progress: {n}/{total} countries fetched...`
   - Collect results into `Map<string, number>` keyed by ISO3
   - Write raw results to `join(rawDir, 'gdelt-raw.json')` as `{ fetchedAt, results: Object.fromEntries(map) }`
   - Convert to `RawIndicator[]` with `indicatorName: 'gdelt_instability'`, `source: 'gdelt'`, `year: currentYear`
   - Write parsed output to `join(rawDir, 'gdelt-parsed.json')` as `RawSourceData`
   - Return `FetchResult` with success=true and countriesFound count

7. Fallback: wrap main logic in try/catch. On failure, use `findLatestCached('gdelt-parsed.json')` pattern identical to gdacs.ts.

**Important constraints:**
- Do NOT use Promise.all or parallel fetching -- GDELT rate limits are undocumented
- The 500ms delay is critical to avoid being throttled
- Use indicator name `gdelt_instability` (not gdelt_stability -- it measures instability, which is inverse)
- Each country request should have its own try/catch so one failure does not abort all
