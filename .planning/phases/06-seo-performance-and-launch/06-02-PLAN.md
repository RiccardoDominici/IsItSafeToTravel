**Mobile viewport fixes:**

1. In `src/pages/en/index.astro` and `src/pages/it/index.astro`, change the map section height from:
   `style="height: calc(100vh - 4rem);"` to `style="height: calc(100dvh - 4rem);"`
   This prevents mobile browser chrome (address bar/toolbar) from cutting off the map. The `dvh` unit accounts for dynamic viewport changes on mobile Safari/Chrome.

2. In `src/components/SafetyMap.astro`, audit the container dimensions. If the map uses `100vh` anywhere in its inline script or styles, change to `100dvh`. Add a CSS fallback for browsers without dvh support using `@supports`:
   ```css
   /* In the component's style block */
   .map-container { height: 100vh; } /* fallback */
   @supports (height: 100dvh) { .map-container { height: 100dvh; } }
   ```

**Search mobile fixes:**

3. In `src/components/Search.astro`, audit the search dropdown/results panel. Ensure:
   - The results dropdown has `class="... w-full sm:w-80 ..."` (full-width on mobile, fixed width on desktop)
   - Each search result item has `min-height: 44px` via class `min-h-[44px]` for touch targets
   - Add `py-2 px-3` padding on each result for comfortable touch spacing
   - The search toggle button itself should be at least 44x44px: add `min-w-[44px] min-h-[44px]` if not already

**Touch target audit:**

4. In `src/components/Header.astro`, verify all clickable elements (nav links, language switcher, dark mode toggle, mobile menu button) have minimum 44x44px touch targets. Add `min-h-[44px] min-w-[44px] inline-flex items-center justify-center` to any that are smaller.

5. In `src/styles/global.css`, add at the top level:
   ```css
   /* Ensure minimum body text size for iOS (prevents auto-zoom on input focus) */
   html { font-size: 16px; }

   /* Touch target minimum */
   a, button, [role="button"], summary, input, select, textarea {
     min-height: 44px;
   }

   /* Ensure links in flowing text don't get forced to 44px height */
   p a, li a, span a, td a {
     min-height: auto;
   }
   ```

**Font loading optimization:**

6. In `src/layouts/Base.astro`, add font preload hints in `<head>` BEFORE the fontsource CSS imports. Find the actual font file paths by checking `node_modules/@fontsource-variable/inter/files/` and `node_modules/@fontsource-variable/dm-sans/files/`. Add preload for the latin subset woff2 files only (not all unicode ranges):
   ```html
   <!-- Font preload for critical rendering path -->
   <link rel="preload" href="/_astro/inter-latin-wght-normal.*.woff2" as="font" type="font/woff2" crossorigin />
   <link rel="preload" href="/_astro/dm-sans-latin-wght-normal.*.woff2" as="font" type="font/woff2" crossorigin />
   ```
   NOTE: The exact filenames with hashes are generated at build time. The executor must run `npx astro build` first to find the actual hashed filenames in `dist/_astro/`, then use those exact paths. If the hashed filenames cannot be determined reliably, SKIP the preload step -- font-display: swap already handles this acceptably and incorrect preload paths cause double-loading which hurts performance.

**Build verification:**

7. Run `npx astro build` to ensure all changes compile. Check the build output for any warnings about deprecated CSS, missing files, or bundle size issues. Verify the sitemap XML files exist in `dist/` and contain hreflang annotations.
