Create src/pipeline/fetchers/reliefweb.ts following the acled.ts pattern exactly. Key implementation details:

1. Export `fetchReliefweb(date: string): Promise<FetchResult>` as the main function.

2. API endpoint: `https://api.reliefweb.int/v2/disasters`
   - Query parameters: `filter[field]=status&filter[value]=current` to get active disasters only
   - Add `fields[include][]=country` to get country associations
   - Add `fields[include][]=type` to get disaster type
   - Add `fields[include][]=status` to confirm active status
   - Add `fields[include][]=glide` for unique disaster ID
   - Set `limit=1000` to get all active disasters
   - Set header `Accept: application/json`
   - Set header with appname: use `process.env.RELIEFWEB_APPNAME || 'isitsafetotravel.com'`

3. No auth required (appname is recommended courtesy, not required).

4. Parse response: ReliefWeb returns `{ data: [{ id, fields: { country: [{ iso3, name }], type: [{ name }], status } }] }`.
   - Count active disasters per country ISO3 code (countries come with ISO3 natively from the API -- use `fields.country[].iso3`).
   - Produce one RawIndicator per country with `indicatorName: 'reliefweb_active_disasters'`, `source: 'reliefweb'`, value = count of active disasters for that country.
   - Set `year: new Date().getFullYear()`, `fetchedAt` to current ISO string.

5. Write raw API response to `reliefweb.json` in getRawDir(date).
6. Write parsed RawSourceData to `reliefweb-parsed.json` in getRawDir(date).

7. Fallback: On any error (network, parse, timeout), call `findLatestCached('reliefweb-parsed.json')`. If found, read it, write to current date dir, return success with error note. If not found, return failure. Use `AbortSignal.timeout(30_000)` for the fetch call.

8. Log pattern: `[RELIEFWEB] Fetching active disaster data...`, `[RELIEFWEB] Successfully processed data for N countries (M indicators)`.

Create src/pipeline/fetchers/gdacs.ts following the acled.ts pattern. Key implementation details:

1. Export `fetchGdacs(date: string): Promise<FetchResult>` as the main function.

2. API endpoint: `https://www.gdacs.org/gdacsapi/api/events/geteventlist/SEARCH`
   - Query parameters: `alertlevel=Red;Orange` to filter to significant events only
   - Add `eventlist=EQ;TC;FL;VO` for earthquake, tropical cyclone, flood, volcano
   - Add `country=` (empty -- get all countries, filter after)
   - Set header `Accept: application/json`
   - Use `AbortSignal.timeout(30_000)` for the fetch call.

3. No auth required.

4. Parse response: GDACS returns `{ features: [{ properties: { alertlevel, eventtype, country, iso3, fromdate, todate, ... } }] }` (GeoJSON-like format).
   - For each feature, extract `properties.iso3` (3-letter country code). If `iso3` is missing or empty, try `properties.country` with `getCountryByName()` to resolve ISO3.
   - Only count events where `properties.alertlevel` is "Red" or "Orange" (case-insensitive).
   - Count qualifying alerts per country ISO3 code.
   - Produce one RawIndicator per country with `indicatorName: 'gdacs_disaster_alerts'`, `source: 'gdacs'`, value = count of active orange/red alerts for that country.
   - Set `year: new Date().getFullYear()`, `fetchedAt` to current ISO string.

5. Write raw API response to `gdacs.json` in getRawDir(date).
6. Write parsed RawSourceData to `gdacs-parsed.json` in getRawDir(date).

7. Fallback: Same as ReliefWeb -- findLatestCached('gdacs-parsed.json') on any error.

8. Log pattern: `[GDACS] Fetching disaster alerts (orange/red)...`, `[GDACS] Successfully processed data for N countries (M indicators)`.

9. IMPORTANT: The GDACS API can be slow or return XML instead of JSON in some cases. If response content-type is not JSON, log a warning and fall back to cache. Also handle the case where the API returns an empty features array (no active alerts) -- this is valid, write an empty indicators array.
