**Sub-fetcher 1: Austria (BMEIA) -- HTML-07 -- HIGHEST CONFIDENCE:**
- URL: `https://www.bmeia.gv.at/reise-services/reisewarnungen`
- Strategy: Fetch page, extract `bmeiaCountrySecurityInfos` JS object via regex: `/bmeiaCountrySecurityInfos\s*=\s*(\{[\s\S]*?\});/`
- Parse the JSON object. Keys are ISO2 country codes, values have `security` (1-4) and `securityPartial` (0 or partial level).
- For each entry: `level = Math.max(entry.security, entry.securityPartial || 0)`, clamped to 1-4.
- Map ISO2 to ISO3 via `getCountryByIso2(iso2)`.
- Direct 1-4 mapping (no normalization needed, already unified scale).
- Indicator name: `advisory_level_at`, source: `advisories_at`.
- Advisory info key: `at`, source text: `Austrian Federal Ministry`.

**Sub-fetcher 2: France (diplomatie.gouv.fr) -- HTML-01:**
- Strategy: Build a French name -> CountryEntry map from `COUNTRIES` using `country.name.fr`.
- Fetch RSS feed at `https://www.diplomatie.gouv.fr/spip.php?page=backend_fcv` to get country list.
- Parse RSS XML with cheerio to extract `<item>` entries. Each has `<title>` (French country name) and `<link>` (URL to country page).
- Batch-crawl per-country pages (concurrency 5, be polite) to extract advisory color/level.
- On each country page, use cheerio to find advisory text/elements. Look for color keywords: rouge, orange, jaune, vert. Use `normalizeFrColor()`.
- If per-country crawl fails to find level text, look for CSS classes or image references containing color names.
- Match French country name from RSS `<title>` to our country data using the fr name map.
- Indicator name: `advisory_level_fr`, source: `advisories_fr`.
- Advisory info key: `fr`, source text: `French Ministry of Foreign Affairs`.

**Sub-fetcher 3: Hong Kong (Security Bureau OTA) -- HTML-05:**
- URL: `https://www.sb.gov.hk/eng/ota/`
- Strategy: Fetch OTA page, use cheerio to find all country links (pattern: `/eng/ota/note-*.html` or `/eng/ota/info-*.html`).
- Extract alert level from main page content. Look for CSS classes `yellowAlert` (Amber) and `redAlert` (Red), or blanket alert text.
- If blanket alert exists (e.g., "Red Outbound Travel Alert is issued for all overseas countries"), set all listed countries to that level.
- Otherwise, try fetching individual country pages to get per-country alert.
- Use `normalizeHkAlert()` for level mapping.
- Country names in English, use `getCountryByName()`.
- Indicator name: `advisory_level_hk`, source: `advisories_hk`.
- Advisory info key: `hk`, source text: `Hong Kong Security Bureau`.

**Sub-fetcher 4: New Zealand (SafeTravel.govt.nz) -- HTML-02:**
- Strategy: The main destinations page may be JS-rendered. Try fetching `https://www.safetravel.govt.nz/destinations` first.
- If HTML contains country data, parse it with cheerio.
- If empty (JS-rendered), try individual country pages: build slug list from COUNTRIES English names (lowercase, spaces to hyphens), fetch `https://www.safetravel.govt.nz/destinations/{slug}`.
- Look for level text: "Exercise normal precautions", "Exercise increased caution", "Avoid non-essential travel", "Do not travel".
- Use `normalizeIeRating()` (same 4-level text pattern) or direct text matching.
- Concurrency 5 for per-country pages.
- Indicator name: `advisory_level_nz`, source: `advisories_nz`.
- Advisory info key: `nz`, source text: `New Zealand SafeTravel`.
- IMPORTANT: If the site is fully JS-rendered and inaccessible, log warning and return empty result (graceful degradation).
