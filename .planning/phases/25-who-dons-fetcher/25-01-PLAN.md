    Create `src/pipeline/fetchers/who-dons.ts` exporting `fetchWhoDons(date: string): Promise<FetchResult>`.

    **API call:**
    - GET `https://www.who.int/api/news/diseaseoutbreaknews`
    - Accept: application/json
    - 30-second timeout via AbortSignal.timeout
    - No auth required

    **Parsing logic:**
    - The API returns a JSON object with a `value` array of DON items
    - Each item has `Title`, `PublicationDateGMT` (or similar date field), and possibly region/country fields
    - Filter to items published within the last 90 days from `date` parameter
    - Extract country from the title field using the pattern "Disease - Country" (split on " - " and take the last segment, trim whitespace)
    - Some titles may have multi-country format or sub-regions -- try getCountryByName on the extracted segment; skip if no match
    - Also try common patterns like "Disease in Country" (split on " in " if dash split fails)
    - Count qualifying outbreaks per country ISO3

    **Output:**
    - Write raw API response to `{rawDir}/who-dons.json`
    - Write parsed data to `{rawDir}/who-dons-parsed.json` as RawSourceData with:
      - source: "who-dons"
      - indicators with indicatorName: "who_active_outbreaks", value: count per country
      - year: currentYear, source: "who-dons"

    **Error handling (follow gdacs.ts pattern exactly):**
    - try/catch wrapping entire fetch+parse
    - On failure: call findLatestCached('who-dons-parsed.json')
    - If cached data found: write it to current rawDir, return success with error note
    - If no cache: return { source: 'who-dons', success: false, countriesFound: 0, error }

    **Logging:**
    - `[WHO-DONs] Fetching disease outbreak news...`
    - `[WHO-DONs] Found {N} DONs in last 90 days`
    - `[WHO-DONs] Successfully processed data for {N} countries ({M} indicators)`
    - On error: `[WHO-DONs] Fetch failed: {error}`
    - On cache fallback: `[WHO-DONs] Using cached data from {path}`
