1. Change `timeout-minutes: 20` to `timeout-minutes: 30` on line 22 (inside the `pipeline` job).

2. Replace the "Check pipeline output" step with an enhanced version that:
   - Checks if `data/scores/latest.json` exists
   - Reads country count from the JSON using node
   - Warns (using `::warning::`) if country count is below 100 (expected 180+)
   - Reads the pipeline log for source success count if available
   - Outputs a GitHub Actions job summary using `$GITHUB_STEP_SUMMARY` with:
     - Date of pipeline run
     - Number of countries scored
     - A warning line if count is low
   - Fails the step (exit 1) only if no output file exists at all

Replace the "Check pipeline output" step `run:` block with:
```bash
if [ -f "data/scores/latest.json" ]; then
  echo "Pipeline produced output"
  COUNTRY_COUNT=$(node -e "const d=JSON.parse(require('fs').readFileSync('data/scores/latest.json','utf8')); console.log(d.countries.length)")
  echo "Countries scored: $COUNTRY_COUNT"

  # Write job summary
  echo "## Daily Pipeline Results" >> $GITHUB_STEP_SUMMARY
  echo "- **Date:** $(date -u +%Y-%m-%d)" >> $GITHUB_STEP_SUMMARY
  echo "- **Countries scored:** $COUNTRY_COUNT" >> $GITHUB_STEP_SUMMARY

  if [ "$COUNTRY_COUNT" -lt 100 ]; then
    echo "::warning::Only $COUNTRY_COUNT countries scored (expected 180+)"
    echo "- **Warning:** Low country count — some sources may have failed" >> $GITHUB_STEP_SUMMARY
  fi
else
  echo "::error::Pipeline did not produce output"
  echo "## Daily Pipeline Results" >> $GITHUB_STEP_SUMMARY
  echo "- **Status:** FAILED — no output produced" >> $GITHUB_STEP_SUMMARY
  exit 1
fi
```

Keep all other steps (checkout, setup-node, install, run pipeline, copy scores, commit+push) exactly as they are. Do NOT change the cron schedule, permissions, env vars, or continue-on-error settings.
