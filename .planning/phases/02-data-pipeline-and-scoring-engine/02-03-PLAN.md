
Export `async function runPipeline(dateOverride?: string): Promise<{ success: boolean; countriesScored: number; sourcesSucceeded: number; sourcesTotal: number }>`

Logic:
1. Determine date: `dateOverride || new Date().toISOString().slice(0, 10)` (YYYY-MM-DD format)
2. Log: `"=== Pipeline run for {date} ==="`
3. Load weights config: `readJson<WeightsConfig>('src/pipeline/config/weights.json')` — abort if missing
4. **Stage 1: Fetch** — Call `fetchAllSources(date)`, collect FetchResult array
5. Log fetch summary: how many succeeded / total
6. **Stage 2: Load raw data** — Read all `*-parsed.json` files from `data/raw/{date}/` into a `Map<string, RawSourceData>`. Each fetcher writes `{source}-parsed.json` containing a RawSourceData object.
7. If no raw data available at all (all fetchers failed and no cached data): exit with code 1 and log error
8. **Stage 3: Score** — Call `computeAllScores(rawDataMap, weightsConfig)`
9. Log: "Scored {N} countries"
10. **Stage 4: Snapshot** — Call `writeSnapshot(date, scoredCountries, fetchResults, weightsConfig.version)`
11. Log: `"=== Pipeline complete: {N} countries scored, {M}/{T} sources succeeded ==="`
12. Return result object
13. Set process exit code: 0 if all sources succeeded, 1 if some failed, 2 if all failed

Add auto-execution when run directly (not imported):

Create `.github/workflows/data-pipeline.yml`:

```yaml
name: Daily Data Pipeline

on:
  schedule:
    # Run daily at 06:00 UTC
    - cron: '0 6 * * *'
  workflow_dispatch:
    # Allow manual triggering from GitHub UI
    inputs:
      date:
        description: 'Date override (YYYY-MM-DD, leave empty for today)'
        required: false
        type: string

permissions:
  contents: write  # Needed to commit data files back to repo

jobs:
  pipeline:
    runs-on: ubuntu-latest
    timeout-minutes: 15

    steps:
      - name: Checkout repository
        uses: actions/checkout@v4
        with:
          token: ${{ secrets.GITHUB_TOKEN }}

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: npm

      - name: Install dependencies
        run: npm ci

      - name: Run data pipeline
        run: npx tsx src/pipeline/run.ts ${{ github.event.inputs.date || '' }}
        env:
          ACLED_API_KEY: ${{ secrets.ACLED_API_KEY }}
          ACLED_EMAIL: ${{ secrets.ACLED_EMAIL }}
        continue-on-error: true
        id: pipeline

      - name: Check pipeline output
        run: |
          if [ -f "data/scores/latest.json" ]; then
            echo "Pipeline produced output"
            COUNTRY_COUNT=$(node -e "const d=JSON.parse(require('fs').readFileSync('data/scores/latest.json','utf8')); console.log(d.countries.length)")
            echo "Countries scored: $COUNTRY_COUNT"
          else
            echo "::warning::Pipeline did not produce output"
            exit 1
          fi

      - name: Commit and push data
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          git add data/raw/ data/scores/
          if git diff --cached --quiet; then
            echo "No data changes to commit"
          else
            DATE=$(date -u +%Y-%m-%d)
            git commit -m "data: update safety scores for $DATE

            Automated daily pipeline run.
            See data/scores/$DATE.json for details."
            git push
          fi

      - name: Trigger site rebuild (optional)
        if: success()
        run: |
          echo "Data committed. The push above will trigger the deploy.yml workflow automatically."
```

Key design decisions in the workflow:
- `continue-on-error: true` on pipeline step so the commit step still runs even with partial failures (some sources may be unavailable)
- `permissions: contents: write` allows committing data back to the repo
- `workflow_dispatch` allows manual triggering with an optional date override for backfilling
- `timeout-minutes: 15` prevents runaway API calls from consuming Actions minutes
- The push to main in the commit step automatically triggers the existing deploy.yml workflow (site rebuild)
- ACLED credentials passed as secrets (env vars)
